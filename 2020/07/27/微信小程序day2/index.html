发表于 2020-07-27 分类于 前端

模板语法

插值表达式

书写位置

  • 标签之间
  • 标签属性
  • 模板语法中
  • 任何使用到js代码或变量的地方都需要使用双重花括号

条件判断

列表渲染

wx:for

  • 应用场景
    • 数组
    • 对象
    • 数值
    • 字符串
  • wx:for = "

wx:for-index

自定义接受数组索引的变量

wx:for-item

自定义接受数组索引的元素

wx:key

  • 在逻辑层和界面层建立一一对应关系，方便局部更新
  • 简单数组
    • wx:key="*this"
    • 数组的元素都是基本数据类型
  • 列表数组
    • {users:[{id:1,name:'zs'},{id:2,name:'ls'}]}
    • <view wx:for=""></view>
    • wx:key="不重复的属性" , 例如 wx:key:"id", 这里不用写花括号

默认变量

  • index 数组索引
  • item 数组元素

hidden属性

通过样式来控制一个元素的实现状态

可选值

  • true 隐藏
  • false 显示

模板的引用

import

  • 定义 <template name="header"><view>公共头部</view></template>
  • 导入 <import src="./header.wxml"/>
  • 调用 <template is="header"></template>

include

  • 定义 header.wxml <view>公共头部</view>
  • 导入 <include src="./header.wxml/>

事件绑定

bind+事件名

存在事件冒泡

catch+事件名

不存在事件冒泡

常见事件名称

  • 点击事件 tap
  • input输入框和textarea文本域中 input

wxss

rpx

  • 响应式的像素单位
  • 1rpx~=0.5px
  • rpx比px更精细

单项数据流

双向数据绑定

逻辑层-界面层
小程序中没有实现双向数据绑定

必须通过事件来实现

this.setData()

  • 更新逻辑层的数据
  • 通知界面层进行更新
